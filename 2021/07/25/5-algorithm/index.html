算法学习

1. 算法导论

1.1 算法分析

1.1.1 数学模型

  1. 近似

    使用高阶函数复杂度代替函数复杂度的方式

    使用~f(N)来表示所有随着N的增大除以f(N)的结果趋近于1的函数

  2. 示例

    • f(N)=N^3^/6-N^2^/2+N/3
    • ~f(N)=N^3^/6
    • 使用~f(N)近似代替f(N)的方式

  3. 增长数量级

    N^3^ /6-N^2^/2+N/3 的增长数量级为 O(N^3^)。增长数量级将算法与它的具体实现隔离开来，一个算法的增长数量级为 O(N^3^) 与它是否用 Java 实现，是否运行于特定计算机上无关

  4. 内循环

    执行最频繁的指令决定了程序执行的总时间，把这些指令称为程序的内循环

  5. 成本模型

    使用成本模型来评估算法，例如数组的访问次数就是一种成本模型

1.1.2 注意事项

  1. 大常数

    在求近似时，如果低级项的常数系数很大，那么近似的结果是错误的

  2. 缓存

    计算机系统会使用缓存技术来组织内存，访问数组相邻的元素会比访问不相邻的元素快很多

  3. 最坏情况下的性能保证

    在核反应堆、心脏起搏器或者刹车控制器中的软件，最坏情况下的性能是十分重要的

  4. 随机化算法

    通过打乱输入，去除算法对输入的依赖

  5. 均摊分析

    将所有操作的总成本除于操作总数来将成本均摊

1.1.3 64选4

  1. 背景

    64匹马,8个赛道,找出跑得最快的4匹马

  2. 常规分析

    1. 将马分为8组，每组各跑一次，然后淘汰掉每组的后四名

    2. 对上面8组的第一名进行一次单独跑，淘汰掉本轮后四名所在的组，并淘汰掉D2，D3，D4，C3，C4，B4；此时还剩下10匹马，A1是最快的，所以剩下9匹马

    3. 余下的马再跑一次选出前三并和A2在跑一次

  3. 极限分析

    1. 先跑一组选第一
    2. 将上次第一递归与下组7匹跑直到选出前四

1.1.4 三数和

用于统计一个数组中和为0的三元组数量

  1. 一般解法

    public int threeNumSum(int[] array) {
    // 获取数组长度
    int length = array.length;
    // 初始化计数器
    int count = 0;
    // 拿出第一个数字
    for (int i = 0; i < length - 1; i++) {
        // 拿出第二个数字
        for (int j = i + 1; j < length; j++) {
            // 拿出第三个数字
            for (int k = j + 1; k < length; k++) {
                // 计算三个数字的和
                int sum = array[i] + array[j] + array[k];
                // 如果三数和为0，则计数器自增
                if (sum == 0) {
                    count++;
                }
            }
        }
    }
    return count;
}

  2. 双指针解法—适用于有序数组

    先对数组进行升序排序，在使用双指针进行遍历查找，时间复杂度为O(N^2)

    public int threeNumSum(int[] array) {
    // 对数组进行排序，升序
    Arrays.sort(array);
    // 获取数组长度
    int length = array.length;
    // 初始化计数器
    int count = 0;
    // 遍历数组，拿到每一个值索引，当左指针值小于右指针值时就一直查找
    for (int i = 0; i < length - 2; i++) {
        // 初始化左指针索引
        int leftpointer = i + 1;
        // 初始化右指针索引
        int rightpointer = length - 1;
        // 初始化目标值
        int target = -array[0];
        while (leftpointer < rightpointer) {
            // 计算左右指针值的和
            int sum = array[leftpointer] + array[rightpointer];
            // 如果目标值等于左右指针值的和，计数器自增，左指针前移，右指针后移
            if (target == sum) {
                count++;
                leftpointer++;
                rightpointer--;
            }else if (sum < target) {
                // 如果左右指针值的和小于目标值，则只对左指针进行前移
                leftpointer++;
            }else {
                // 如果左右指针值的和大于目标值，则只对右指针进行后移
                rightpointer--;
            }
        }
    }
    return count;
}

1.1.5 倍率计算

  1. 假设T(N) ~ aN^b^log2^N^，那么T(2N)/T(N) ~ 2^b^

  2. 例如对于暴力的ThreeSum算法，近似时间为~N^3^/6；进行如下实验，多次运行该算法，每次取的N值为前一次的两倍，统计每次执行的时间，并统计本次运行时间与前一次运行时间的比值，得到如下结果

    N Time(ms) Ratio
    500 48 /
    1000 320 6.7
    2000 555 1.7
    4000 4105 7.4
    8000 33575 8.2
    16000 268909 8.0

    可以看到，T(2N)/T(N) ~ 2^3^，因此可以确定 T(N) ~ aN^3^logN

2. 排序算法

  1. 概述

    Java主要排序方法为java.util.Arrays.sort()，对于原始数据类型使用三向切分的快速排序，对于引用类型使用归并排序

    假定排序元素需要实现 Java 的 Comparable 接口，该接口有 compareTo() 方法，可以用与判断两个元素的大小关系。

    使用辅助函数 less() 和 swap() 来进行比较和交换的操作，使得代码的可读性和可移植性。

    排序算法的成本模型是比较和交换的次数。

  2. 接口定义

    public abstract class Sort<T extends Comparable<T>> {

    public abstract void sort(T[] nums);

    // 值比较
    protected boolean less(T before, T after) {
        return before.compareTo(after) < 0;
    }

    // 值交换
    protected void swap(T[] atarget, int before, int after) {
        T temp = atarget[before];
        atarget[before] = atarget[after];
        atarget[after] = temp;
    }
}

2.1 选择排序

  1. 概述

    选择数组中最小元素，将它与数组的第一个元素交换位置

    再从数组剩下的元素中选择出最小的元素，将它与数组的第二个元素交换位置

    重复操作，直至整个数组有序

  2. 时间复杂度分析

    选择排序需要 ~N^2^/2 次比较和 ~N 次交换，它的运行时间与输入无关

  3. 接口实现

    public class Selection<T extends Comparable<T>> extends Sort<T> {
    // 对指定数组进行排序
    @Override
    public void sort(T[] nums) {
        // 获取数组长度，避免频繁访问数组
        int length = nums.length;
        for (int i = 0; i < length - 1; i++) {
            int min = i;
            for (int j = i + 1; j < length; j++) {
                if (less(nums[j], nums[min])) {
                    min = j;
                }
            }
            swap(nums, i, min);
        }
    }
}

  4. 普通实现

    public int[] sort(int[] nums) {
    // 获取数组长度，避免频繁访问数组
    int length = nums.length;
    // 初始化临时变量，用于值交换
    int temp;
    // 初始化最小索引
    int minIndex;
    for (int i = 0; i < length - 1; i++) {
        // 假定当前值最小
        minIndex = i;
        // 在剩下的元素中找是否有比当前值还小的元素，找出其索引
        for (int j = i + 1; j < length; j++) {
            if (nums[j] < nums[minIndex]) {
                minIndex = j;
            }
        }
        // 值交换，保证nums[i]始终是最小值
        temp = nums[i];
        nums[i] = nums[minIndex];
        nums[minIndex] = temp;
    }
    return nums;
}

2.2 冒泡排序

  1. 概述

    从左到右不断交换相邻逆序的元素，在一轮的循环之后，可以让未排序的最大元素上浮到右侧

  2. 时间复杂度分析

    最差情况下需要交换(n-1)!次

  3. 接口实现

    public class Bubble<T extends Comparable<T>> extends Sort<T> {

    @Override
    public void sort(T[] nums) {
        int length = nums.length;
        boolean isSorted = false;
        for (int i = length - 1; i > 0 && !isSorted; i--) {
            isSorted = true;
            for (int j = 0; j < i; j++) {
                if (less(nums[j + 1], nums[j])) {
                    isSorted = false;
                    swap(nums, j, j + 1);
                }
            }
        }
    }
}

  4. 普通实现

    public int[] sort(int[] nums) {
    // 获取数组长度，避免频繁访问数组
    int length = nums.length;
    // 初始化临时变量，用于值交换
    int temp;
    // 如果相邻两数之间前者比后者大就交换两数位置
    for (int i = 0; i < length; i++) {
        for (int j = i + 1; j < length; j++) {
            if (nums[i] > nums[j]) {
                temp = nums[i];
                nums[i] = nums[j];
                nums[j] = temp;
            }
        }
    }
    return nums;
}

2.3 插入排序

  1. 概述

    每次都将当前元素插入到左侧已经排序的数组中，使得插入之后左侧数组依然有序

    插入排序每次只能交换相邻元素，令逆序数量减少 1，因此插入排序需要交换的次数为逆序数量

  2. 时间复杂度分析

    平均情况下插入排序需要 ~N^2^/4 比较以及 ~N^2^/4 次交换；
    最坏的情况下需要 ~N^2^/2 比较以及 ~N^2^/2 次交换，最坏的情况是数组是倒序的；
    最好的情况下需要 N-1 次比较和 0 次交换，最好的情况就是数组已经有序了。

  3. 接口实现

    public class Insertion<T extends Comparable<T>> extends Sort<T> {
    @Override
    public void sort(T[] nums) {
        int length = nums.length;
        for (int i = 1; i < length; i++) {
            for (int j = i; j > 0 && less(nums[j], nums[j - 1]); j--) {
                swap(nums, j, j - 1);
            }
        }
    }
}

  4. 普通实现

    public int[] sort(int[] nums) {
    // 获取数组长度，避免频繁访问数组
    int length = nums.length;
    // 初始化临时变量，用于值交换
    int temp;
    for (int i = 1; i < length; i++) {
        for (int j = i; j > 0; j--) {
            if (nums[j] < nums[j - 1]) {
                temp = nums[j];
                nums[j] = nums[j - 1];
                nums[j - 1] = temp;
            }
        }
    }
    return nums;
}

2.4 希尔排序

  1. 概述

    希尔排序对间隔 h 的序列进行排序。通过不断减小 h，最后令 h=1，就可以使得整个数组是有序的

  2. 时间复杂度分析

    N 的若干倍乘于递增序列的长度

  3. 接口实现

    public class Shell<T extends Comparable<T>> extends Sort<T> {
    @Override
    public void sort(T[] nums) {

        int length = nums.length;
        int gap = 1;

        while (gap < length / 3) {
            gap = 3 * gap + 1; // 1, 4, 13, 40, ...
        }

        while (gap >= 1) {
            for (int i = gap; i < length; i++) {
                for (int j = i; j >= gap && less(nums[j], nums[j - gap]); j -= h) {
                    swap(nums, j, j - gap);
                }
            }
            gap = gap / 3;
        }
    }
}

  4. 普通实现

    private static int[] sort(int[] nums) {
    // 获取数组长度，避免频繁访问数组
    int length = nums.length;
    int gap = 1;
    int temp;

    // 确定初始增量
    while (gap < length / 3) {
        gap = 3 * gap + 1; // 1, 4, 13, 40, ...
    }

    // 增量小于等于1就一直进行值交换，等于1为冒泡排序
    while (gap >= 1) {
        for (int i = gap; i < length; i++) {
            for (int j = i; j >= gap; j -= gap) {
                // 比较增量序列大小，按升序排列
                if (nums[j] < nums[j - gap]) {
                    temp = nums[j];
                    nums[j] = nums[j - gap];
                    nums[j - gap] = temp;
                }
            }
        }
        // 缩小增量
        gap = gap / 3;
    }
    return nums;
}

2.5 归并排序

2.5.1 归并方法

public abstract class MergeSort<T extends Comparable<T>> extends Sort<T> {
    // 初始化辅助数组
    protected T[] aux;

    protected void mergeSort(T[] nums, int leftPosition, int rightPosition, int length) {
        int leftPointer = leftPosition;
        int rightPointer = rightPosition + 1;

        for (int k = leftPosition; k <= length; k++) {
            // 将数据复制到辅助数组
            aux[k] = nums[k];
        }

        for (int k = leftPosition; k <= length; k++) {
            if (leftPointer > rightPosition) {
                nums[k] = aux[rightPointer++];
            } else if (rightPointer > length) {
                nums[k] = aux[leftPointer++];
            } else if (aux[leftPointer].compareTo(aux[rightPointer]) <= 0) {
                // 先进行这一步，保证稳定性
                nums[k] = aux[leftPointer++];
            } else {
                nums[k] = aux[rightPointer++];
            }
        }
    }
}

    public class Up2DownMergeSort<T extends Comparable<T>> extends MergeSort<T> {

    private void sort(T[] nums, int l, int h) {
        if (h <= l) {
            return;
        }
        int mid = l + (h - l) / 2;
